- Desvantagens: linha de comando, comandos crpticos, d medinho no comeo

Conceitos bsicos (por alto):

  • Repositrio: lugar onde o cdigo vive. O repositrio padro uma pasta no seu computador. Repositrios em outros computadores so chamados de remote. Cada remote recebe um nome e tem um endereo prprio. Por conveno, se existe um repositrio remoto central, ele ser chamado de origin

  • Commit: uma verso salva do seu cdigo, com nome, nmero, e nome de quem fez, acompanhado de uma breve descrio de o que mudou (commit message). Pode ser de um s arquivo ou de vrios.

  • Diff: comando que compara as diferenas entre commits (verses) do cdigo, linha por linha. Serve para ver o que mudou de uma verso para outra

  • Merge: quando duas verses do mesmo arquivo esto em conflito (merge conflict), o Git pede para voc fundir (merge) as duas, escolhendo quais linhas sero mantidas na verso final. a parte mais chata do Git

  • Status: comando que te informa o esstado atual do seu repositrio local, se houve alguma alterao em algum arquivo, e se ele est incronizado com o remote

  • Pull e Push: pull copia os commits de um repoitrio remoto (remote) para seu repositrio local. push envia os commits do seu local para o remote. Enviar para o remote garante um backup atualizado fora da sua mquina

  • Branch: um conjunto de verses (commits) que formam uma linha do tempo separada de outros branchs. Geralmente usado quando se quer testar jeitos diferentes de fazer algo, ou verses experimentais. O branch padro o master, e cada branch tem seu nome

  • Clonar: copiar um repositrio existente

  • Init: criar um repositrio novo

  • .gitignore - arquivo com os nomes e tipos de arquivo que voc quer que o git ignore

Cada commit tem um RG, chamado de commit ID, que nos permite comparar verses dos arquivos, ou reverter para uma verso anterior. Ele tambm contm um ponteiro para o commit imediatamente anterior a ele. O conjunto dos commits, ligados atravs de seus ponteiros, permite reconstruir a histria do repositrio, e navegar por ela

Seu repositrio como uma filogenia de arquivos: desde a origem, cada modificao fica registrada no ramo da rvore (cada commit um fssil), e o txon terminal do ramo o estado atual do repositrio. Se quisermos reverter para um estado anterior, s indicar qual fssil queremos acessar:

git show FOSSIL:arquivo > arquivoMedroso cria um arquivo chamado arquivoMedroso, como o conteudo de arquivono commit FOSSIL. s abrir arquivoMedroso e copiar e colar o que quiser pra pra arquivo, e dar um comit novo, e depois apagar arquivoMedroso

CUIDADO: isso muda todo o contedo do arquivo pra verso gravada como FOSSIL, tudo que for alterado entre FOSSIL e o estado atual perdido.

O ramo original (sempre chamado de master por conveno) pode ser visto como a verso release, final ou estvel do cdigo: todo o desenvolvimento e experimentaes ocorre em branches, e quando estiver tudo funcionando, as alteraes so passadas para o branch master

Merge ou fuso de ramos

transferncia horizontal de genes entre ramos da rvore filogentica

git mergepega o ltimo commit de um branch e une ele com outro branch. Isto pode gerar conflitos se houverem verses do mesmo arquivo que tenham a mesma linha com contedos diferentes.

Para fazer o merge, puxamos o HEAD de um branch pro outro. HEAD o ltimo commit do branch:

# puxando de master para outro branch
git merge master

git mergejunta os arquivos linha por linha. Se no houver conflito, o git junta tudo sem falar nada. Se a mesma linha for diferente nas duas verses do arquivo, o git marca a regio de conflito da seguinte forma:

<<<<<<< HEAD
texto branch atual
=======
texto outro branch
>>>>>>> arquivo

Para corrigir conflitos de merge, altere o arquivo mantendo a parte que desejar, e d um novo commit. O git no deixa voc alterar nada at que o conflito seja resolvido

Tudo que fizemos agora foi no repositrio local, ou seja, no seu prprio computador. Git descentralizado, isto , cada pessoa tem seu repositrio local, onde pode fazer suas alteraes, e pode enviar essas alteraes para um repositrio externo.
